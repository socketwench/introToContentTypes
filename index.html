Different Types, Different Data

Different Fields

When we categorize content, we can immediately see that different types contain different data. This picture node is very different from this page node. The picture has an image file and some metadata on how to display the image. The page, on the other hand, just as a body of text to display.

But What About These Two?

Okay, that's easy, but what about these two? This blog post and this page node both have a title and body. The fields are the same. So what makes them different?

Different Types, Different Uses

Different ways it's displayed or managed

Well, these two are different not because of different fields, but their use case. That is, the different ways they are displayed or managed. For a blog post, you generally want to know the author that wrote it. For a page -- like an "About Us" page -- the author isn't important. So even while the page has an author, we don't display it to visitors. The different ways we use content also makes something a different type.

What is a Content Type Made Of?

When creating a content type, the site builder can also define zero or more custom fields. There's a lot of different kinds of fields you can use, including rich content for the body field.

What is a Content Type Made Of?

Permissions are also associated with the content type -- not specific pieces of content. These permissions grant the ability to create, edit, or delete nodes of that type.

What is a Content Type Made Of?

Lastly, content types also provide key metadata. This includes the unique type itself, allowing you to build custom filters for your content.
